## 二、详细差距分析

### 🔴 高优先级（阻塞性问题）

#### 2.1 ASCII 编码错误（Bug #3）

**问题描述**：`@天气 成都` → `'ascii' codec can't encode characters in position 5-6: ordinal not in range(128)`

**根因分析**：
- 中文消息在某些路径下被以 ASCII 编码处理
- 可能出现在：`urllib.request` 的 `data` 参数编码、或 MCP 工具调用参数传递中
- 需要全局排查所有涉及字符串编码的位置

**涉及文件**：
- `app/controllers/user_chat.py` — SSE 流式对话、数字员工调用
- `app/controllers/admin_employee.py` — 后台员工调用
- `app/mcp/tools.py` — MCP 工具参数传递
- `app/services/collector.py` — 采集请求编码

**修复方案**：全局确保所有 `str.encode()` 使用 `'utf-8'` 编码，排查 `urllib.request.Request` 的 data 参数

#### 2.2 @数字员工卡片渲染（任务6.2 #1）

**问题描述**：后台配置了天气卡片模板，但前台 @天气 返回后没有渲染成卡片样式

**根因分析**：
- 前端已有 `event: card` SSE 事件处理 → `renderInfoCard()` 函数
- 但后端未正确发送 `card` 事件，或卡片 JSON 格式与前端期望不一致
- 天气 API 员工（API型）的返回没有经过卡片渲染模板处理

**涉及文件**：
- `app/controllers/user_chat.py` — `UserEmployeeInvokeHandler._invoke_api_employee()`
- `app/templates/user_chat.html` — `renderInfoCard()` 函数
- `app/models/digital_employee.py` — `response_render_template` 字段

**修复方案**：
1. 后端 API 型员工调用后，按 `response_render_template` 格式化 → 发送 `event: card` SSE 事件
2. 前端实现 `renderInfoCard()` 渲染函数（天气卡片、新闻卡片等模板）
3. 确保 `event: employee` 事件发送员工**名称**而非编码

#### 2.3 对话消息缺少元信息（任务6.2 #2）

**问题描述**：对话气泡下方需要显示响应时间(s) 和 Token 大小

**当前状态**：

- `appendMessage()` 已支持 tokens 和 elapsed 参数 ✅
- SSE `event: stats` 事件后端已发送 ✅

**差距**：

**修复方案**：
1. `UserEmployeeInvokeHandler` 末尾补齐 `event: stats` SSE 事件
2. `loadConversation()` 中显示历史消息的 token_count

#### 2.4 智能问数数据命中率低（任务6.3 #1）

**问题描述**：教师测试发现成都相关数据只有12条，查询关键字详细匹配不足，无法按意图分析

**根因分析**：

2. 扩充瞭望源模板（增加更多新闻源）

#### 2.5 ECharts 报表呈现增强（任务6.3 #3/4）

### FIX-4: FTS5 全文搜索

**`app/models/data_warehouse.py`** 新增方法:
```python
@staticmethod
def search_fts(keyword: str, limit: int = 10) -> list:
    """使用 FTS5 全文索引搜索（优先），回退 LIKE。"""
    with get_db() as conn:
        # FTS5 查询语法
        fts_query = ' OR '.join(f'"{w}"' for w in keyword.split())
        try:
            rows = conn.execute(
                "SELECT dw.* FROM data_warehouse dw "
                "JOIN data_warehouse_fts fts ON dw.id = fts.rowid "
                "WHERE data_warehouse_fts MATCH ? "
                "ORDER BY rank LIMIT ?",
                (fts_query, limit)
            ).fetchall()
        except:
            rows = []
        if not rows:
            # 回退 LIKE
            rows = conn.execute(
                "SELECT * FROM data_warehouse WHERE title LIKE ? OR summary LIKE ? "
                "ORDER BY created_at DESC LIMIT ?",
                (f"%{keyword}%", f"%{keyword}%", limit)
            ).fetchall()
        return [dict(r) for r in rows]
```
