# Issue 修复要求

## 核心原则

1. **深入分析问题的根因**，从源头修复问题，而不是逃避或者用其他补丁掩盖掉
2. 分析完问题的根因后开始高质量修复

## 每次修改完代码后，必须确保进行以下步骤：

1. **手动测试所有相关功能是否正常**，必须严格测试到功能，而不是仅仅测试编译通过或者可以成功访问页面，一定要测试到具体功能
2. **再次检查这次修改是否带来新的 bug 和问题**
3. **更新 README 和 docs 内相关文档**
4. **更新 test 文件夹内的内容**

## 其他规则

- 修改前先理解项目架构和代码流程
- 保持最小变更原则，不改与 issue 无关的代码
- 遵循项目已有的代码风格和命名约定
- 所有修改必须有清晰的注释说明
